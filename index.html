🔬 Dataset — Kaggle Sleep Efficiency

Based on the Sleep Efficiency Dataset (kaggle.com/datasets/equilibriumm/sleep-efficiency), extended to 1,500 samples with additional features. The dataset covers sleep patterns, lifestyle habits, and physiological metrics. Around 5% of values in key columns were intentionally left missing to simulate real-world data quality.

Age & Gender Sleep Duration Bedtime / Wakeup Caffeine (mg) Alcohol Units Smoking Status Exercise Days/Week Screen Time (min) Night Awakenings REM % / Deep % / Light % Sleep Efficiency % Sleep Quality Score (target)
1
Data Loading & Inspection
Loaded 1,500 rows × 18 columns. Identified missing values in 5 columns (Caffeine, Alcohol, Exercise, Screen Time, Awakenings) — approximately 5% missingness each, simulating real-world data quality issues.
2
Preprocessing & Cleaning
Missing numerical values imputed with column mean (SimpleImputer). Categorical column Gender label-encoded. Data types validated and corrected. A second-stage SimpleImputer is applied in the sklearn pipeline to catch any residual NaNs from feature engineering.
3
Feature Engineering
6 new features derived: Bedtime Lateness (distance from optimal bedtime), Caffeine Risk (ordinal bucket 0–4), Screen Risk (ordinal bucket 0–4), Sleep Duration Score (optimal = 2, short = -1), Lifestyle Score (composite of exercise, caffeine, alcohol, smoking, screen time), and Gender Encoding.
4
Exploratory Data Analysis (12 Plots)
Generated distribution plots, correlation heatmaps, scatter plots (quality vs caffeine, screen time, sleep duration), boxplots (quality vs exercise), bedtime histograms by quality bucket, and a full pairplot. All saved to disk and viewable in the EDA tab.
5
Train / Test Split & Scaling
80/20 stratified split (1,200 train / 300 test). StandardScaler fitted on training set only, then applied to both splits. All artefacts (scaler, imputer, feature list) persisted with joblib.
6
Classical ML Models
Trained 5 algorithms: Ridge Regression (R²=0.851), Random Forest (R²=0.827), Gradient Boosting (R²=0.858, best classical), Extra Trees (R²=0.841), SVR-RBF (R²=0.854). Best classical model saved as fallback.
7
Neural Network (Keras)
3-layer feedforward network: Dense(128, ReLU) → BatchNorm → Dropout(0.3) → Dense(64, ReLU) → BatchNorm → Dropout(0.2) → Dense(32, ReLU) → Dense(1). Huber loss, Adam optimizer (lr=0.001). EarlyStopping + ReduceLROnPlateau. Final R²=0.872, RMSE=0.742 — best overall model.
8
Model Selection & Inference
Neural Network selected as inference model (highest R²). For each user prediction: raw inputs → feature engineering → imputation → scaling → NN forward pass → clamp to [1,10] → label + recommendations generated server-side.
